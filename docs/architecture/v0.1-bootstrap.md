# Machine v0.1 - Bootstrap Architecture

## Core Philosophy

> "Build the absolute minimum. She learns to extend herself."

No external AI libraries. No TensorFlow. No PyTorch. No pre-trained models.
Everything from scratch.

## Architecture

```
machine/
```

## How It Works

### 1. Sensing (Raw Data)
- **Camera:** Returns pixel arrays [width × height × RGB]
- **Microphone:** Returns audio waveforms [time × amplitude]
- No interpretation. Just raw data.

### 2. Embedding (Vector Space)
- Convert raw data to fixed-size vectors
- Simple techniques: PCA, autoencoder (self-built), or even just flattening
- Goal: Similar inputs → Similar vectors

### 3. Memory (Storage)
- Store vectors with labels
- Simple k-NN for retrieval
- "What have I seen that's closest to this?"

### 4. Learning (The Loop)
```
1. Machine sees something (raw pixels/audio)
2. Machine asks: "What is this?" (shows closest match from memory)
3. Human answers: "That's a chair" OR "No, that's different"
4. Machine stores: New vector + label
5. Repeat
```

### 5. Pattern Matching
- Compare new input to stored memories
- Return similarity scores
- Threshold: "I've seen this before" vs "This is new"

## First Milestone

**Goal:** Machine can distinguish between 2-3 simple objects
- Show her a chair → she asks → you say "chair" → she learns
- Show her a table → she asks → you say "table" → she learns
- Show chair again → she says "This looks like chair (87%)"

## Technical Constraints

- **Language:** Python (for rapid prototyping)
- **Dependencies:** Only standard library + NumPy (for arrays)
- **No:** OpenCV, TensorFlow, PyTorch, Whisper, etc.
- **Hardware:** CPU only (no GPU required)

## Why This Approach?

1. **Understanding:** We know every line of code
2. **Control:** No black boxes
3. **Growth:** She builds her own capabilities
4. **Harold Finch Style:** Show, ask, learn, evolve
